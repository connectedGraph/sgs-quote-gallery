# 数据来源说明

`db/line.json` 的全部内容来自 **三国杀 BWIKI**（<https://wiki.biligame.com/sgs/>）的公开 MediaWiki API。本文记录：

1. 当前实际使用的 API 端点与现状（截至 2026-05 实测）
2. `line.json` 是怎么一步步生成的
3. 后续如何调 API 拉取/补充最新台词

---

## 一、用到的 API 端点（GPT-5.5 Thinking 逐项验证）

API 入口：`https://wiki.biligame.com/sgs/api.php`，`GET` 请求，建议 ≤1 req/s。

### 1. 搜索页面 — `action=query&list=search`

用于把"武将名/皮肤名"映射到 wiki 页面标题。

```
GET https://wiki.biligame.com/sgs/api.php
    ?action=query
    &list=search
    &srsearch=许劭
    &srlimit=20
    &format=json
    &utf8=1
```

实测返回 `query.search[]`，每项含 `title` / `pageid` / `snippet` / `timestamp`。例：搜 `许劭` 命中 105 条，第一条是武将主页 `许劭`，还会带出 `烽火连天*许劭`、`评世雕龙*许劭` 等皮肤页。

> 状态：**可用**。`title` 字段可直接传入下面两个接口，不需要额外编码。

### 2. 取 wikitext 源码 — `action=query&prop=revisions`

用于读"皮肤模板"原文（`{{皮肤|...}}`），里面带 `|台词=` 字段，结构稳定但里面的台词分隔符自己一套语法（如 `;` 分技能、`/` 分句），解析成本不低。

```
GET https://wiki.biligame.com/sgs/api.php
    ?action=query
    &titles=烽火连天*许劭
    &prop=revisions
    &rvprop=content|timestamp
    &rvslots=main          ← 新版 Bwiki 必加，否则没有内容
    &format=json
    &utf8=1
```

内容在 `query.pages.<pageid>.revisions[0].slots.main["*"]`。

> 状态：**可用**。注意：旧写法 `revisions[0]["*"]` 在新 Bwiki 上拿不到。

### 3. 取渲染后 HTML — `action=parse&prop=text`

旧的 `db/数据工具/Bwiki-html转json/main.py` 走的就是这条路，但它**不能直接消费 API 返回**，原因见下。

```
GET https://wiki.biligame.com/sgs/api.php
    ?action=parse
    &page=许劭
    &prop=text
    &format=json
    &utf8=1
```

HTML 在 `parse.text["*"]`。

> 状态：**可用**，但有重大警告 ↓↓↓

#### ⚠️ HTML 里的音频链接不是真链接

API 返回的 `<span class="bikit-audio">` 里 `data-src="01,2#许劭,评荐"` 是占位编码，不是 mp3 URL。真实 mp3 由 wiki 页面里的 JS 函数 `TranToPinYin(".wj.bikit-audio","data-src")` 在浏览器中拼出来，拼完后类名也会从 `bikit-audio` 变成 `bikited-audio`。

也就是说：

| 数据流 | `data-src` 内容 | class 名 |
|---|---|---|
| 直接 `action=parse` 取的 HTML | `"01,2#许劭,评荐"` 这种占位符 | `bikit-audio` |
| 浏览器打开页面、JS 跑完之后另存的 HTML | `https://web.sanguosha.com/.../XuShao_PingJian_01.mp3` | `bikited-audio` |

`db/数据工具/Bwiki-html转json/main.py` 里 `span[class*='bikited-audio']` 这个选择器，**只能匹配后者**。所以这条历史链路是"浏览器渲染 → 另存 HTML → 喂给 Tk 工具"，不是纯 API。

## 二、`line.json` 是怎么生成的（历史链路）

逐皮肤、半手工：

1. **搜索**：用 BWIKI 站内搜索或上面的搜索 API 找到目标皮肤页（如 `评世雕龙*许劭`）。
2. **浏览器渲染**：在浏览器中打开该页面，等待页面 JS 跑完，使每个 `<span class="bikit-audio">` 被改写为 `bikited-audio` + 真实 mp3 链接。
3. **另存 HTML**：F12 复制元素，或保存网页源码到本地 `.html`。
4. **跑 GUI 工具**：执行 `python db/数据工具/Bwiki-html转json/main.py`，在 Tk 窗口里：
   - 填"英雄名 = 许劭"、"皮肤名 = 评世雕龙"
   - "读取 HTML 文件"加载第 3 步的文件
   - 点"生成 JSON"，工具用以下选择器解析：
     - `div.flex-container[style*='gap: 5px;']` — 一项技能
     - 内部 `div.basic-info-row-label` — 技能名
     - 内部 `span[class*='bikited-audio']` — 取 `data-src` 作为 mp3 URL
   - 输出形如 `{ hero, skin, skills:[{ name, quotes:[{ text, audio, explanation:"" }] }] }`，并把"阵亡"技能强制排到末尾。
5. **拼装 line.json**：把每个皮肤的 JSON 块手工/脚本追加进 `db/line.json` 数组。
6. **后处理**（也在 `db/数据工具/`）：
   - `阵亡技固定末置/main.py` — 兜底再排一次"阵亡"。
   - `无效audio-url清除/main.py` — 把 404/坏链 audio 字段清空，使前端的 `<AudioButton>` 走 disabled 态。
7. **解析（explanation）字段**：JSON 里所有 `explanation` 是后续单独由 LLM 生成的 markdown，不来自 wiki，所以 wiki API 不负责这一项。

## 三、推荐的 API 调用方式（拉取最新台词）

如果只想增量补一个新皮肤的台词，**走"wikitext + 静态 mp3 路径"** 比走 HTML 链路省事得多，因为：

- wikitext 里 `|台词=` 字段是结构化的，纯文本好分段。
- mp3 文件名在 sanguosha.com 走"皮肤代号 + 技能拼音 + 序号"的固定规则（`web.sanguosha.com/10/pc/res/assets/runtime/voice/skin/<code>/<Hero>_<Skill>_<NN>.mp3`），可以从已有 line.json 里反推同皮肤其它句子。

### 推荐流程

```python
import requests, time

API = "https://wiki.biligame.com/sgs/api.php"
UA  = {"User-Agent": "sgs-quote-sync/1.0"}

def search(keyword, limit=10):
    r = requests.get(API, params={
        "action": "query", "list": "search",
        "srsearch": keyword, "srlimit": limit,
        "format": "json", "utf8": 1,
    }, headers=UA, timeout=15)
    time.sleep(1)
    return r.json()["query"]["search"]

def get_wikitext(title):
    r = requests.get(API, params={
        "action": "query", "titles": title,
        "prop": "revisions",
        "rvprop": "content|timestamp",
        "rvslots": "main",                # 必填
        "format": "json", "utf8": 1,
    }, headers=UA, timeout=15)
    time.sleep(1)
    pages = r.json()["query"]["pages"]
    page  = next(iter(pages.values()))
    rev   = page["revisions"][0]
    return rev["slots"]["main"]["*"], rev["timestamp"]
```

### 要踩的坑

1. **`*` 不用转义**。`titles=烽火连天*许劭` 直接传，requests 会替你 urlencode。
2. **NFC 警告无害**。返回里 `warnings.query` 抱怨 "non-normalized data" 时仍然命中，可忽略；想消掉就在传参前 `unicodedata.normalize("NFC", title)`。
3. **想要真实 mp3 URL 时**，不要从 `action=parse` 的 HTML 里抠。两种可行做法：
   - 复用本仓库 `line.json` 里同皮肤已有的 mp3 路径，按"皮肤代号 + 技能英文名 + 序号"模板替换；
   - 用 headless 浏览器（Playwright / Selenium）打开皮肤页面，等 `bikited-audio` 出现后再 `getAttribute("data-src")`。
4. **频率**。文档建议 ≤1 req/s，批量拉时务必 `time.sleep(1)`，否则会临时被拦。
5. **解析字段不在 wiki 里**。`explanation` 是台词逐句鉴赏，由 LLM 后补，不在 BWIKI API 覆盖范围。
